# Lore Constraints

This document is the place to record narrative truths, terminology, cosmology, and setting rules that implementation and visual design should respect.

This file should evolve collaboratively.

## Purpose

Use this document to capture:

- canonical entities
- world-history anchors
- philosophical distinctions
- terminology that must stay precise
- narrative taboos
- presentation implications for the UI

## Current Known Constraints

### Apollo Matters

Apollo is foundational to the universe and should be treated as a meaningful absence even if not yet represented directly in the original codebase.

Questions for future clarification:

- Was Apollo visible to ordinary people, or primarily infrastructural?
- Is Apollo remembered as a system, a governance presence, a mythic absence, or all three?
- Should traces of Apollo in the UI feel sacred, central, missing, deprecated, or forbidden?

### Elpis And Void Must Not Collapse Into Simple Morality

They should not be reduced to good and evil palette swaps. Their differences are ideological, structural, and experiential.

### The World Should Explain Itself Through Contact

Narrative delivery should prefer implication, system behavior, wording, evaluation posture, environmental text, and consequence over exposition dumps.
